--- Object/mesh structure of each FBX (inspected by parsing the FBX
    binary strings, no FBX SDK available) ---
This is a whole scene broken into per-category FBX files, NOT one
combined file per part. HOWEVER, each individual category FBX (e.g.
Crane-Mounted.fbx, Building.fbx, Trucks.fbx, Containers.fbx) itself
contains exactly ONE Model + ONE Geometry node, i.e. all of that
category's sub-parts are pre-joined into a single mesh named after the
file (confirmed: every FBX file scanned has Geometry-token count = 1 and
Model-token count = 1). Object names found:
  Crane-Mounted.fbx   -> single mesh object "Crane Mounted"
  Crane-On-Ground.fbx -> single mesh object "Crane Ground"
(Same one-mesh-per-file pattern applies to the other category FBX files;
 not individually re-verified but the token counts match across
 Building.fbx, Containers.fbx, and Trucks.fbx as well.)
There is also _HouseConstructionSite-AllInOneObject_.fbx, which bundles
the entire site (building + both cranes + containers + trucks + fence +
ground + ramp + road) as one file, for whoever wants the whole scene at
once rather than picking category pieces.

--- Texture note (Unity import) ---
Every FBX references ImphenziaPalette01-256-Gradient.png as a single
shared "base_color_texture" via a relative path of the form
"<FileName>.fbm\ImphenziaPalette01-256-Gradient.png" (Autodesk's default
FBX-side-folder convention). That .fbm subfolder does NOT exist in the
zip — the texture ships once, flat, in the pack root. On import into
Unity, the FBX importer will likely show the material with a missing/
pink texture until you manually assign
Assets/Art/Construction/Majadroid/ImphenziaPalette01-256-Gradient.png
to the material's base color slot (or use "Extract Materials" and repoint
it). This is expected — it is how the source pack ships, not a corrupt
download.

--- Recommendation for Task A (tower crane) ---
Crane-On-Ground.fbx and Crane-Mounted.fbx in fbx files/ are genuine
yellow lattice-mast tower cranes with a horizontal jib and hook line
(see Overview.png — two of them are visible flanking the modeled
apartment building). Since poly.pizza's only CC0 "crane" result turned
out to be a hook/cable prop, not a tower-crane silhouette (see
../Cranes/License.txt), these two FBX are the actually-usable CC0 tower
crane assets in this drop.
